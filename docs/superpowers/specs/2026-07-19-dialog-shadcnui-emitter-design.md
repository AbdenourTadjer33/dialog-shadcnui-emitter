# dialog-shadcnui-emitter — v1 Design

Date: 2026-07-19
Status: Approved

## Purpose

An open-source, shadcn-style registry that distributes a reusable "imperative dialog" system:
a performance-focused draggable dialog built on shadcn/ui conventions (Radix + Tailwind + `cn`),
driven by a typed mitt event emitter so any code can open/close dialogs without declaring
`<Dialog>` components at every call site. Replaces copy-pasting these files between projects.

Consumption model: `npx shadcn@latest add <github-pages-url>/r/<item>.json` — code is copied
into the consuming project (shadcn philosophy: you own the code).

## Out of scope for v1

- The "dialog registry" variant for large projects (register dialog components by key,
  `openDialog('user-edit', props)`). Planned for a later iteration.
- npm package distribution.
- Automated interaction tests (drag behavior is verified manually in the demo playground).

## Registry items

| Item | Files installed | npm deps | registry deps |
|---|---|---|---|
| `use-drag-transform` | `hooks/use-drag-transform.ts` | — | — |
| `draggable-dialog` | `components/draggable-dialog.tsx` | `@radix-ui/react-dialog`, `lucide-react` | shadcn `utils` (`cn`) |
| `dialog-manager` | `components/dialog-manager.tsx`, `lib/dialogs.ts` | `mitt` | `use-drag-transform`, `draggable-dialog`, shadcn `button` |

Installing `dialog-manager` pulls the entire system via registryDependencies. Projects that
only want the declarative draggable dialog (no emitter) install `draggable-dialog` alone.

## Components

### `hooks/use-drag-transform.ts` (existing — unchanged)

Transform-only drag hook: drag layer ref + drag handle ref, pointer-capture gesture,
bounds clamped to viewport (measured once per gesture), keyboard arrow-key movement,
`allowOverflow`, `margin`, `keyboardStep` options, `resetPosition()`. No changes.

### `components/draggable-dialog.tsx` (existing — one change)

Composable shadcn-style parts: `DraggableDialog`, `DraggableDialogTrigger`,
`DraggableDialogClose`, `DraggableDialogOverlay`, `DraggableDialogContent` (portal +
overlay + centering container + transform-only drag layer), `DraggableDialogHeader`
(drag handle), `DraggableDialogBody`, `DraggableDialogFooter`, `DraggableDialogTitle`,
`DraggableDialogDescription`.

Change: `closeLabel` default `"Fermer"` → `"Close"`.

### `lib/dialogs.ts` (new)

The public imperative API. Replaces the project-specific `@/lib/emitter` and
`@/services/dialogs/dialog.types` imports.

- `type DialogModalSize = 'xs' | 'sm' | 'md' | 'lg' | 'xl' | '2xl' | '3xl' | '4xl' | '5xl' | 'full'`
- `type DialogCustomBody = (close: () => void) => React.ReactNode`
- `type DialogAgreeCallback = (close: () => void) => void | Promise<void>`
- `type DialogDisagreeCallback = () => void`
- `type OpenDialogInput` — all optional: `title`, `message`, `description` (alias for
  `message`, message wins), `customBody`, `showActions`, `modalSize`,
  `modalContentClassName`, `isDraggable`, `allowOverflow`,
  `options: { agree: string; disagree: string }`, `agree`, `disagree`
- `type DialogEvents = { 'open-dialog': OpenDialogInput; 'close-dialog': void }`
- `const dialogEmitter = mitt<DialogEvents>()` — typed; the `@ts-expect-error`
  workarounds in the manager are removed
- `openDialog(input?: OpenDialogInput)` / `closeDialog()` — callable from anywhere,
  no React required

### `components/dialog-manager.tsx` (existing — reworked)

Singleton component mounted once at app root. Subscribes to `dialogEmitter` on mount
(passing the handler to `emitter.off` so only its own listener is removed), renders the
draggable dialog from state, always reopens centred (`resetPosition()` before open).

Changes from current code:

- Imports from `@/lib/dialogs` (typed emitter + types) instead of `@/lib/emitter` /
  `@/services/dialogs/dialog.types`.
- English defaults, configurable via a new optional `labels` prop:
  `{ title?, message?, agree?, disagree?, close? }` merged over
  `{ title: 'Are you sure?', message: 'Please confirm that you want to proceed. This action may be irreversible.', agree: 'Confirm', disagree: 'Cancel', close: 'Close' }`.
  Per-call `OpenDialogInput` fields still override per dialog. `close` is passed as
  `closeLabel` to `DraggableDialogContent`.
- Everything else (size classes map, agree/disagree flow, custom body with
  sr-only description) stays as-is.

## Repo structure

```
registry/                     # registry source files (shadcn-style layout)
  hooks/use-drag-transform.ts
  components/draggable-dialog.tsx
  components/dialog-manager.tsx
  lib/dialogs.ts
registry.json                 # item definitions (names, files, deps)
public/r/*.json               # generated by `npx shadcn build`
demo/                         # Vite playground; manual drag testing + usage example; not deployed
.github/workflows/deploy.yml  # on push to main: build registry, publish to GitHub Pages
README.md                     # install commands, API docs, examples
docs/superpowers/specs/       # this document
```

The current root-level `dialog-manager.tsx`, `draggable-dialog.tsx`, `use-drag-transform.ts`
move into `registry/` (reworked as above).

## Build, CI, deploy

- `npx shadcn build` generates `public/r/*.json` from `registry.json`.
- GitHub Actions on push to main: install, `tsc --noEmit`, `shadcn build`, deploy
  `public/` to GitHub Pages.
- Stable install URLs: `https://<owner>.github.io/dialog-shadcnui-emitter/r/<item>.json`.

## Verification

- Type-check (`tsc --noEmit`) in CI.
- Registry build succeeds in CI (validates `registry.json` and file references).
- Manual: run the demo playground; verify open/close via `openDialog()`/`closeDialog()`,
  drag with pointer and keyboard, bounds clamping, `allowOverflow`, `full` size,
  custom body, labels override, and a fresh `shadcn add` into a scratch project.
